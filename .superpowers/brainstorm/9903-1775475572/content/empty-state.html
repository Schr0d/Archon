Empty state text when no graph is loaded

What should users see when they first open the viewer? Needs warmth and a clear next step.

A

"No graph loaded yet. Use File → Open to visualize your dependencies."

Pros: Clear action, helpful tone, tells users exactly what to do.

Cons: Assumes users know how to get a graph file.

B

"Welcome to Archon. Open a dependency graph to explore your codebase structure."

Pros: Friendly welcome, explains the tool's purpose.

Cons: More abstract, less direct guidance.

C

"Open a graph file to begin"

Pros: Short, clear, gets straight to the point.

Cons: Less warmth, doesn't explain what the viewer does.
